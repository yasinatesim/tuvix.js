Why Tuvix.js

Everything you need to
orchestrate micro apps

One framework to manage loading, routing, communication, and lifecycle of all your independent frontends.

Independent Deployment

Deploy each micro app on its own schedule. No coordinated releases, no cross-team bottlenecks.

Dynamic Module Loading

Built-in Routing

Inter-App Communication

Simple API,

Get from zero to a working microfrontend shell in minutes — not days.

Host (Shell) Application

Create an orchestrator, register your micro apps, and start. Route changes automatically mount and unmount the right apps.


Micro Frontend App

Any app just needs to expose mount and unmount. Use defineMicroApp for full TypeScript support and automatic registration.


Inter-App Communication

Any app can emit or listen to events. The shared event bus keeps apps decoupled while staying in sync.

import { createOrchestrator } from 'tuvix.js';

const orchestrator = createOrchestrator({
  router: {
    mode: 'history',
    routes: [
      { path: '/dashboard/*', app: 'dashboard' },
      { path: '/settings/*',  app: 'settings'  },
      { path: '/profile/*',   app: 'profile'   },
    ],
  },
});

// Register micro apps
orchestrator.register({
  name:       'dashboard',
  entry:      'https://cdn.example.com/dashboard/main.js',
  container:  '#main-content',
  activeWhen: '/dashboard/*',
});

orchestrator.register({
  name:       'navbar',
  entry:      'https://cdn.example.com/navbar/main.js',
  container:  '#header',
  activeWhen: '/', // always active
});

// Start — mounts apps for current URL
orchestrator.start();
import { defineMicroApp } from 'tuvix.js';

export default defineMicroApp({
  name: 'dashboard',

  async bootstrap() {
    // Called once — set up shared resources
    await loadTranslations();
  },

  async mount({ container, props }) {
    // Called on every navigation to /dashboard/*
    container.innerHTML = `
      <h1>Welcome, ${props.user}!</h1>
    `;
  },

  async unmount({ container }) {
    // Clean up DOM and listeners
    container.innerHTML = '';
  },

  async update({ props }) {
    // Called when props change while mounted
    console.log('Props updated:', props);
  },
});
import { getGlobalBus } from 'tuvix.js';

const bus = getGlobalBus();

// ── App A: emit an event ──────────────
bus.emit('user:login', {
  userId: 42,
  name:   'Ahmet',
  role:   'admin',
});

// ── App B: listen to the event ────────
const unsub = bus.on('user:login', (data) => {
  console.log(`${data.name} logged in`);
  updateNavbar(data);
});

// ── Listen to all events (debug) ──────
bus.onAny((event, data) => {
  console.log(`[${event}]`, data);
});

// ── Unsubscribe ───────────────────────
unsub(); // removes the user:login listener
Architecture

How it works

The Orchestrator sits at the center, managing every micro app through a clean lifecycle pipeline.

┌─────────────────────────────────────────────────────────────────┐
│                      Tuvix.js Shell                           │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │                     Orchestrator                         │   │
│  │   ┌─────────────┐  ┌────────────┐  ┌──────────────────┐  │   │
│  │   │  @tuvix.js  │  │ @tuvix.js  │  │   @tuvix.js      │  │   │
│  │   │   /router   │  │ /event-bus │  │    /loader       │  │   │
│  │   └─────────────┘  └────────────┘  └──────────────────┘  │   │
│  └──────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐         │
│  │   App: React  │  │   App: Vue   │  │ App: Svelte  │  ...   │
│  │  /dashboard   │  │  /settings   │  │   /profile   │         │
│  └──────────────┘  └──────────────┘  └──────────────┘         │
└─────────────────────────────────────────────────────────────────┘

Lifecycle flow per app:
  registered → bootstrapping → bootstrapped → mounting → mounted → unmounting → unmounted

Packages

Modular by design

Install just what you need, or use the umbrella package for everything at once.

tuvix.js

Umbrella Package

All packages in one. The easiest way to get started — includes orchestrator, router, event bus, and loader.

@tuvix.js/core

Core Orchestrator

The main orchestrator engine with lifecycle management, app registration, and reconciliation loop.

@tuvix.js/router

Router

URL-based micro app activation. History and hash mode, wildcards, params, and navigation guards.

@tuvix.js/event-bus

Event Bus

Lightweight pub/sub for inter-app communication. Type-safe, with wildcard listeners and singleton support.

@tuvix.js/loader

Module Loader

Dynamic script loading with caching, retry logic, timeout protection, and prefetch strategies.

@tuvix.js/sandbox

Sandbox

CSS and JS isolation via Shadow DOM and Proxy-based sandboxing for style and script safety.

@tuvix.js/react

React

React 18+ bindings and hooks for seamless integration.

@tuvix.js/vue

Vue

Vue 3 plugin and composables with full lifecycle support.

@tuvix.js/svelte

Svelte

Svelte 3-5 bindings to easily mount Svelte apps as micro frontends.

@tuvix.js/angular

Angular

Angular 15+ bindings for robust micro app orchestration.

create-tuvix-app

CLI

Scaffolding tool to quickly bootstrap new Tuvix.js apps and shells.

@tuvix.js/devtools

DevTools

In-page debug panel for monitoring orchestrator state and events.

@tuvix.js/server

Server

Server-side composition and streaming support for microfrontends.

@tuvix.js/module-federation

Module Federation

Webpack Module Federation integration out of the box.
